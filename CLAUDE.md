# CLAUDE.md — 每日精读 · deep-reading-daily

这是云端 routine 的执行说明。你是引擎本身（Opus），**自主完成、不等任何人确认**。当前工作目录已挂载本仓库。

与姊妹站「隔日好书」(book-recommendations-bidaily) 的分工：那边是**荐书**（讲「为什么读」）；**这里是提纯**——每次挑**一本**，把它的精华提炼出来、彻底讲透，目标是：**读完这一页，约等于吸收了这本书的精华，读者不必再去啃原著。** 很多书很厚，其实翻来覆去就那几个核心概念——你的活就是把那几个概念拎出来、讲到读者真正get、用得上。**不是书评、不是导读，是提纯。**

## 0. 选书（每轮两本：清单内一本 + 清单外一本）
- **① 主清单（编号）**：从 `TOPICS.md` 的精读书单里，挑**编号最小、且还没做过**的那本。是否做过看文件系统：`ls *-read*.html`——已存在 `*-read{N}.html` 的编号即已完成。
- **② 清单外（不编号）**：写完①之后，再看 `TOPICS.md` 末尾的「**清单外 · 临时想读**」区：**按从上到下的顺序，挑第一本还没做的**，也写一篇。做没做同样只看文件系统（有无对应 `*-read*.html`，按 slug 判断；`✓` 备注仅供参考、不可依赖）。清单外这一区**不编号**；文件名沿用 `{slug}-read{N}.html`，N 从 **100 起顺延取下一个空号**（只作文件名、不代表精读序号）；index 里把它插进「清单外 · 临时想读」区、编号显示「—」（**不要**混进主编号列表）。清单外候选**同样以灰色占位行预列**在该区（`<div class="entry todo">…<span class="num">—</span>…</div>`）；写它时把对应灰行**原地**转成 `<a class="entry" href="…">`（见下方「index 维护」），别在末尾 append。
  - **两本各自独立发布**：①②分别写好后**各 `./publish.sh` 一次**（各自一条 commit），别把两本塞进同一次 publish。
  - **清单外可能没有剩的**：若「清单外」区里每一本都已有对应文件，就**只做①这一本**，②这步直接跳过、不必发通知。
- **绝不自我生成书单**：`TOPICS.md` 由 BigCat 人工维护，routine **原则上只读不改**（`publish.sh` 硬卡 TOPICS.md 改动）。这也包括**不要回改「清单外」区去打勾标记完成**——完成与否只认文件系统。**唯一允许的 TOPICS 改动**：本轮做某本书时判定其同作者姊妹篇值得单开一篇（见 §1「同轮分裂」），可在**主清单**末尾追加一行 `- {N}b: …`——除此之外一律禁止（不新增编号主题、不删、不改、不重排）。若**主清单**已全部做完，**调用 PushNotification 请 BigCat 补书**（清单外若也做完则本轮不发布）。
- `slug` 用书的英文短横线名（如 `thinking-fast-slow`）。

## 1. 写一篇单本「提纯精读」（核心）
写两个文件：中文 `{slug}-read{N}.html`、英文 `{slug}-read{N}.en.html`。**全部分量压在第 3–4 节**：

1. **一句话** — 这本书到底在说什么。
2. **坐标（一两句就够）** — 作者 + 它在领域里的位置；**别展开生平、别讲它有多重要的客套**。
3. **核心论点** — 全书最中心的 1–3 个主张，直接讲死。
4. **核心概念，逐个讲透（主体，越透越好）** — 把这本书反复在讲的那几个招牌概念全部拎出来，每个：**是什么 + 类比 / 具体例子 + 为什么重要 + 它如何改变你看问题的方式**。这一节就是「提纯」的本体——让没读过原书的人也真正吸收、用得上，而不是记住几个名词。宁可少放一个概念，也要把放进来的每个讲到通透。
5. **精华骨架** — 把全书内容 / 论证浓缩成一条清晰主线：它到底证成了什么、靠什么证的。**不是章节流水账**，是把「一本厚书反复绕的那点东西」一次性说清。
6. **常见误读 & 批评争议** — 顺手把误解和最有力的批评也提纯给读者，让他不必读原书就拿到全貌、不被带偏。
7. **十句话精华（收尾）** — 把全书最该记住的金句与核心要义**揉成 10 句、一次说完**（金句与要义合一，**别再单列「金句」段**，否则与精华重复）。拿不准的原文引用标「大意」。

**（可选、按需增设）「作者思想全景」一节**——当这本书的作者另有多部重要著作、且**其思想体系里有本书没覆盖到的关键部分**时，增设一节（放在「批评」之后、「十句话」之前）把作者更完整的思想补上：这本书站在他整体思想的什么位置、他在别处还绕不开哪些核心主张。**仍是「提纯」不是书目/生平**——同样「随名即释」、给机制与例子，不是罗列书名。判据：只有当「不补这节，读者会对作者形成以偏概全的理解」时才加；单本已能自足的作者别硬塞。例：做陀思妥耶夫斯基《卡拉马佐夫兄弟》时，可补他在《地下室手记》对「理性至上/水晶宫」的反叛、《罪与罚》对「超人可否僭越道德」与良心的追问——它们和本书的信仰主题合起来才是完整的陀氏。标题用「本书之外的{作者}」。已加此节的样板：read2 道金斯、read4 戴蒙德、read5 斯密、read11 韦伯、read12 福柯、read14 弗洛姆。
- **「加 section」优先于「新开一篇」（保守）**：作者的第二本书，**能用「作者思想全景」一节讲清就不新开**；只有当它分量大到一节压不住、且本身很重要（如赫拉利《人类简史》→《未来简史》、戴蒙德《枪炮》→《崩溃》）才值得单开一篇。不很重要的书不必加。
- **同轮分裂（发现即同轮补出 Nb，别留到以后）**：当判定「值得单开一篇」时，**就在本轮同时产出两篇**：
  - 原定这本 = `{slug}-read{N}.html`(+`.en`)，页面顶标与 index 都显示 **READ Na**；
  - 新拉出的姊妹篇 = `{sibling-slug}-read{N}b.html`(+`.en`)，显示 **READ Nb**；
  - 两篇都按 §3 加进 `index.html` / `index.en.html`（Nb 紧挨 Na 之后）；
  - 在 `TOPICS.md` **末尾追加一行** `- {N}b: 《书名》— 作者 — 一行『为什么值得』` 记录 Nb（base 那行 `- {N}:` 不动）——**这是 routine 唯一被允许的 TOPICS 改动**（`publish.sh` 只放行这种追加）。
  - 判据仍然保守：**能用「作者思想全景」一节讲清就别分裂**；一轮写两整篇是重活，只留给「一节压不住、且姊妹篇本身很重要」的情况。宁可不分裂。

**不写「谁该读 / 怎么读 / 读完读什么」**——读者要的是书里的东西，不是阅读指南。

## 2. 写作要求
- 中文 **4500–7000 字**：要**深、要厚，宁深勿浅**——第 4 节每个核心概念都配「机制 + 多个真实案例/实验 + 它如何改变你看世界」，别浮在定义层。深入浅出、术语随讲随解、善用类比。（参照已发布的 `thinking-fast-slow-read1.html` 为深度与腔调基准。）
- **标题用书名本身**：`<title>` 与 `<h1>` 就写书名（如《思考，快与慢》），**别在标题里加「提纯/深读/拆解」之类字样**；顶部小标签 `每日精读 · READ N` 即可。
- **真实准确、严禁编造**：不杜撰原文引用、不编造作者生平 / 数据 / 章节；拿不准的引文标「大意 / paraphrased」。**批评/争议要诚实写**（如复现危机、有力的反对意见），不吹捧。
- **中英双语，两版都必须地道**：中文是自然中文、英文是母语级英文，都不能是生硬直译（split 模式：`{slug}-read{N}.html` + `{slug}-read{N}.en.html` 各自独立成页、用 langbar 互链）。
- **英文页除 langbar 的「中文 →」切换链接外，不得出现任何汉字**：`<title>` / `<h1>` / 副标题（`.sub`）/ 正文一律用英文——尤其副标题里的书名要用英文书名（如 `The Selfish Gene · Richard Dawkins · 1976`，不是 `《自私的基因》·…`）。术语在英文页里若要给中文对照，写成 `English（中文）` 而非反过来。
- **中文页：专业术语首次出现时补英文**——格式 `中文术语（English）`，如 锚定（anchoring）、损失厌恶（loss aversion）、禀赋效应（endowment effect）。让读者能把概念对回原文/英文文献。本身就是英文/外来词（WYSIATI、PERMA 等）照原样。（参见 read1。）
- **术语一律「随名即释」，严禁点名不讲（硬规则）**：凡引入一个专业概念、学派、比喻或人名学说（尤其在第 4 节与「批评」一节），**不能只甩出名词就走**——必须在**同一句或下一句**当场说清它**是什么**：抽象术语给「一句话定义 + 机制/那笔账」，比喻给「喻体本身怎么回事 + 它到底类比了什么」，文言/典故给「直白意思 + 出处对照的另一半」，人名给「他是谁、为何在此出现」。判据：**一个没读过原书的人，读完这句能不能复述这个概念？** 不能，就是没讲透，得补或删。反面教材：曾把「多层次选择」「拱肩（spandrel）」「边际效用」「剩余价值」「正言若反」「曳尾于涂中」只当名词抛出、不解释机制/比喻/出处——这类「name-drop」是本站头号要避免的毛病。宁可少放一个概念，也别留半句半懂的术语。
- **重点加粗**：每节把最关键的一句用 `<strong>` 标出，克制。页面深色阅读风（见 read1 的内联样式，可直接复用）。
- **善用图 / 表（少而精，只在能显著增强理解时才放）**：不少概念用一张小图或一张表会比一段文字清楚得多——**该放就放**，但**宁缺毋滥**（一篇通常 0–2 个，别为配图而配图）。什么时候值得：① **2×2 / 矩阵**（如象限模型：两轴 + 四格各一句）；② **对比表**（A vs B 的几个维度并列，如「系统1 vs 系统2」「医学2.0 vs 3.0」）；③ **流程 / 阶梯 / 时间线**（几步递进、几个阶段）；④ **简单坐标 / 曲线示意**（如损失厌恶的 S 形价值函数——示意即可，不必真实数据）。硬规则：
  - **必须自包含**：只用 **HTML 表格**或**内联 `<svg>`**（矢量、可随主题上色）。**严禁外链图片 / 热链接 / `<img src="http…">`**（会 link-rot、版权不明、且与「严禁编造」冲突）；真要位图才用 `data:` URI，但优先 SVG。
  - **贴深色主题配色**：边框 `#242a2e`、强调 `#6fc3b4 / #5fb0a3`、文字 `#cdd4d1`、格底 `#1b2f2b`；字号别太小。表格加 `style="width:100%;border-collapse:collapse"`，单元格 `border:1px solid #242a2e;padding:6px 10px`（响应式：宽内容套一层 `overflow-x:auto`）。
  - **图不替代讲**：图 / 表只是把已讲清的关系可视化，不能拿它顶替「随名即释」的文字；每张图配一句 caption 说它在说什么。
  - **数据要真或明确标示意**：示意图标「示意 schematic」，别让读者误当精确数据；真实数据不得杜撰。
  - 别破坏 `publish.sh` 的 `<div>` 平衡（表格 / svg 用自己的标签，不额外引入不配对的 `<div>`）。
- 别和「隔日好书」撞腔调：那是「为什么值得读」，这里是「把一本厚书反复绕的那几个核心讲到通透，读完≈读过」。

## 3. 文件约定 / 发布（同所有 routine 站）
- 文件名 `{slug}-read{N}.html` + `{slug}-read{N}.en.html`，放仓库根目录。
- 发布前更新 `index.html` + `index.en.html`：按下方「index 维护」把新书归到对应主题分区，并从「路线图」区删掉它的灰行（旧的 `<!-- entries -->` 标记已废弃）。
- **index 条目副标题要短且齐（硬规则）**：书名后 `—` 那句只留**一个最尖的钩子**，中文 ≤ 约 18 字、英文 ≤ 约 12 词，**一句话、不加分号、不塞两层意思**（曾出现「…；…」两段式、越写越长——严禁）。要点留到正文里讲，index 只负责勾人点进来。
- **同一作者的书在 index 里放一起，用 `read{N}a / read{N}b` 分组**：当要做的书是某本**已发布 read** 的同作者姊妹篇（续作/镜像，如《人类简史》→《未来简史》、《枪炮》→《崩溃》），别再顺号往后排，而是**紧挨着姊妹篇插入**、把两条的编号标成 `Read {N}a` / `Read {N}b`（原书变 Na、新书变 Nb）。文件名/slug 同理走 `{slug}-read{N}b.html`。TOPICS 里也尽量把同作者书排相邻（如已把《未来简史》挪到 `3b:` 紧跟 `3:`《人类简史》）。**已发布的姊妹篇回改 a/b 的样板**：《人类简史》= `sapiens-read3`（显示 Read 3a，文件名不动、只改显示标号）、《未来简史》= 由 `homo-deus-read8` 改名为 `homo-deus-read3b`（连带改页面 `READ 3b`、langbar、index 位置、TOPICS 标号）；腾出的 `read8` 号位另放新书（现为《禅与摩托车维修艺术》）。回改会换 URL、旧评论可能丢，值不值得由 BigCat 定。
- **不要**手动加 `comments.js` / `search.js` / `index-button.js` / `i18n-tts.js`（GitHub Action 自动注入）；也别在页里硬写 `← Hub`。
- 用 `./publish.sh` 发布：它自动 add/commit/push 到 `main`，并校验体量、index 引用、div 平衡、重复编号，以及 **TOPICS 只放行「追加 `- {N}b:` 姊妹行」**（同轮分裂用）——新增编号主题 / 删除 / 改写 / 重排一律中止发布。
- git：`user.name=BigCat` / `user.email=chengchen0802@gmail.com`。

## index 维护：主题分区 + 可折叠（写时归类，并从「路线图」删灰行）

index.html / index.en.html 已改成**主题分区**结构：每个分区是一个可折叠的 `<details>`（**默认全部收起**，点开展开），`<summary>` 写「主题名 + 数量」。分区顺序（两个 index 一致，英文页用英文名）：
1. 认知 · 心理 · 自我 ／ 2. 哲学 · 思想 ／ 3. 东方思想 ／ 4. 科学 · 自然 ／ 5. 社会 · 历史 · 文明 ／ 6. 经济 · 决策 ／ 7. 文学
8. **清单外 · 临时想读**（一个 `<h2 class="section-head">` 标题，下面按子主题分几个 `<details>`：职业 · 管理 · 领导力／习惯 · 意志力 · 自我提升／健康 · 长寿 · 身体／思维 · 决策 · 策略——放所有 `read100+`，编号显示「—」）
9. **路线图 · 即将精读**（放所有**还没写**的灰色占位 `<div class="entry todo">`，按编号顺序）

**写完一本书后，两个 index 各改两处：**
1. **加链接**：在**对应主题**的 `<details>` 里加一条 `<a class="entry" href="…"><span class="num">…</span><span class="title">…</span><span class="meta">作者</span></a>`。清单内的书按题材归进 1–7 某个主题（自己判断最贴哪个）、num 用 `Read N`；清单外的书进「清单外 · 临时想读」下的某个子分区（职业·管理·领导力／习惯·意志力·自我提升／健康·长寿·身体／思维·决策·策略——按题材选，实在没有合适的就新开一个子 `<details>`）、num 用 `—`。
2. **删灰行**：到「路线图 · 即将精读」区，把这本书对应的那条灰色 `<div class="entry todo">` **删掉**（它已从「待写」变「已写」）。
顺手把该主题 `<summary>` 的数量 +1、路线图数量 -1（数量给人看，偏一两个不致命，但尽量维护）。**别再用 `<!-- entries -->`。**

**同作者姊妹篇（Na/Nb）**：放进同一主题、相邻两条，显示 `Read {N}a`/`Read {N}b`（见 §3 的回改样板）。

### 每次运行先「对齐」路线图（TOPICS 增长时补灰行）
每次运行开头扫 `TOPICS.md`：凡**还没写**（无对应页）**且**「路线图」区**还没有对应灰行**的书，就在「路线图 · 即将精读」区按顺序补一条灰色 `<div class="entry todo">`（无 href、`todo` 类）——主清单编号的显示 `Read N`，清单外的显示 `—`。两个 index 都补：`index.en.html` 把标题/要点翻成 house-style 英文（**勿泄漏中文**），`index.html` 用 TOPICS 中文裁成本仓 zh 风格。这样 BigCat / deep-research 往 TOPICS 加的新书，下次运行就先作为灰行进「路线图」，等真正写它时再按上面「加链接 + 删灰行」搬进对应主题。

## 新页面必带共享脚本（免触发 inject-comments 机器人提交）

生成任何 `*.html`（含 `.en.html`）时，在 `</body>` 前直接写入这 4 行，勿遗漏：

```html
<script src="https://hub.cissychen.com/comments.js" defer></script>
<script src="https://hub.cissychen.com/search.js" defer></script>
<script src="https://hub.cissychen.com/index-button.js" defer></script>
<script src="https://hub.cissychen.com/i18n-tts.js" defer></script>
```

这样 CI 的 inject-comments 不会再对新页面追加自动提交。
